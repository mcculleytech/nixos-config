Run the pre-merge checklist before merging the current branch to master or before committing to the master branch locally. This ensures documentation stays in sync with completed work.

---

## Step 1: Update README.md

Read `README.md` and identify all TODO/checklist items that have been completed during this session or branch. For each completed item:
- Check it off with today's date in `YYYY-MM-DD` format
- If any new follow-up work was surfaced during this session, add it as a new TODO item

---

## Step 2: Update AUTOMATION_ROADMAP.md

Read `AUTOMATION_ROADMAP.md` and:
- Check off any milestones that are now complete
- Update pipeline diagrams or descriptions if the deployment flow changed

---

## Step 3: Report

Summarize what was checked off and what new items were added in each file. If nothing needed updating in a file, say so explicitly.
